# 04 执行路线与质量门禁

## 1. 目标

将融合结果转化为可执行迭代流程，确保后续文档新增与改动不再出现口径分叉。

## 2. 四阶段路线图

### 阶段 A：口径冻结（1 个迭代）

1. 冻结“训练基线 36 / 平台全量 40”双口径。
2. 明确 `汇总` 目录是否纳入 UTI/AUX 的产品决策。
3. 统一编号策略，以 `doc/02-module-mapping-and-numbering.md` 为准。

### 阶段 B：结构补齐（1-2 个迭代）

1. 补齐缺失的系统索引和交叉链接。
2. 对所有模块执行模板必填项检查。
3. 统一“历史 35”文案为新口径描述。

### 阶段 C：指标落地（2 个迭代）

1. 每个系统至少 1 个模块完成 L1/L2 字段闭环。
2. 样例模块补齐 L3 趋势统计定义。
3. 更新附录中的字段解释，避免同名异义。

### 阶段 D：发布与维护（持续）

1. 发布前执行质量门禁清单。
2. 每次新增模块都触发“索引 + 汇总 + 系列文档”联动更新。
3. 保留变更日志，按日期追踪口径变化。

## 3. 质量门禁（Quality Gates）

### G1 编号门禁

1. 新编号未与现有编号冲突。
2. 所属系统前缀正确。
3. 历史别名已在映射表标注。

### G2 模板门禁

1. 必填章节完整。
2. 参数表与评分表字段可追溯到实现。
3. 结束条件覆盖自然/手动/异常。

### G3 数据门禁

1. L1 字段可落库。
2. L2/L3 计算路径可复现。
3. 无效会话过滤规则已定义。

### G4 索引门禁

1. 系统索引与汇总索引都已更新。
2. 模块链接可达。
3. 统计数字与模块清单一致。

## 4. 变更流程（最小闭环）

1. 修改模块详述文档。  
2. 更新对应系统 `INDEX.md`。  
3. 更新 `glm5/汇总/INDEX.md` 的统计与目录。  
4. 更新 `doc/01` 与 `doc/02` 的口径/映射。  
5. 执行发布前核对清单并记录日期。

## 5. 发布前核对清单（可直接勾选）

- [ ] 模块总数与系统分布和索引一致。  
- [ ] 编号无冲突，新增模块有唯一前缀与序号。  
- [ ] 文档状态枚举值合法（`draft/review/stable`）。  
- [ ] 至少有 L1 指标可落地，且定义清晰。  
- [ ] 本次改动已同步到 `/doc` 系列文档。  

## 6. 当前优先事项（基于 2026-02-27 资料）

1. 明确 UTI/AUX 是否进入 `glm5/汇总` 主索引统计。  
2. 完成 36 个训练模块的统一模板化核验。  
3. 建立跨文档自动校验（编号唯一性、链接有效性、统计一致性）。
