# 斯特鲁普测试 - 详述（讨论稿）

> 文档编号: MOD-ATTENTION-03
> 所属系统: 注意力系统
> 页面: `stroop.html` | 脚本: `stroop.js`
> 返回索引: [GAMES_SUMMARY.md](../../GAMES_SUMMARY.md)

## 1. 关键内容（本轮重点）

- 训练目的: 克服字义干扰，快速识别颜色。锻炼大脑的冲突解决能力与反应速度。
- 能力标签: 抑制控制
- 任务价值: 将行为表现映射为可追踪指标，为后续评级标准建立数据基础。

## 2. 理论出处与意义

- 理论主轴: 注意力系统强调目标选择、干扰抑制与持续警觉。
- 任务意义: 用标准化交互情境采样目标能力，支持训练、评估和纵向比较。

### 2.1 参考出处（首版）

1. Posner & Petersen (1990), attention network model.
2. Treisman & Gelade (1980), feature integration theory.

## 3. 玩法说明

1. 调整参数（如有）并开始任务。
2. 按页面提示完成反应、选择或操作。
3. 系统根据规则给出即时反馈。
4. 达到结束条件后输出结果。

## 4. 规则与胜利条件

- 规则主线: 依据题面目标完成规定操作，错误响应计入统计。
- 完成条件: 完成全部题目、回合或目标。。
- 结束条件: 自然结束 / 手动结束 / 异常中断（待逐模块补齐异常处理口径）。

## 5. 参数说明（默认值与范围）

| 参数ID | 类型 | 默认值 | 取值范围/候选值 |
|---|---|---|---|
| `grid-size` | number | 4 | 3 ~ 7 |
| `play-mode` | radio | manual | manual | voice |

- 注：参数表自动抽取自页面控件，最终口径以模块讨论版定稿为准。

## 6. 评分体系（先定义指标，后定阈值）

- 当前指标建议: 正确率、反应时/完成时长、错误类型分布、稳定性（跨回合波动）。
- 评级状态: 阈值未定稿，等待逐模块讨论后统一发布。
- 讨论入口: 本任务可先确认“核心指标是否充分”，再定 S/A/B/C/D 阈值。

## 7. 辅助内容（后续完善）

1. 数据结构：会话级 + 题目级记录字段。
2. 异常分支：失焦、误触、超时、中断、权限失败。
3. 训练处方：频率、时长、参数递进策略。
4. 参考文献补全：DOI、版次、证据等级。
